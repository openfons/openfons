# 平台版开源可商用技术选型方案

> 版本：v1.2  
> 日期：2026-03-12  
> 适用：长期采集、清洗、分析、结构化展示（TanStack Table）
> 文档角色：执行摘要（SoT 见 `docs/plan/plan0/平台级采集与分析架构方案（开源可商用版）.md`）
> 冲突裁决：与 SoT 不一致时，以 SoT 为准。

---

## 1. 目标

在“人手充足”的前提下，直接建设中长期平台架构，避免以脚本为核心反复重构。  
目标是形成可持续运行、可扩展、可商用的开源技术底座。

---

## 2. 选型原则

1. 优先开源且可商用（MIT/BSD/Apache/PostgreSQL License 优先）。  
2. 平台优先，业务逻辑适配层自研。  
3. 数据分层（Raw/Silver/Gold）和证据链必须从 Day 1 设计。  
4. 合规优先：只采公开数据，遵守 robots 与平台 API 条款。  
5. 生产链路禁止依赖临时脚本：`collect.mjs/clean.mjs` 仅作为迁移期回填与验证工具。  

---

## 3. 平台级推荐架构（主推荐）

```text
Airflow（唯一编排真源）
  -> Crawlab（执行网关与运行管理）
    -> HTTP Worker + Browser Worker（Pinchtab）
      -> Valkey + Kafka（去重/队列/事件流）
        -> SeaweedFS（Raw 原始证据）
        -> PostgreSQL（元数据/事务）
        -> ClickHouse（分析）
        -> OpenSearch（检索）
          -> dbt + Great Expectations（建模与质量）
            -> API 服务层
              -> TanStack Table 前端
```

## 5. 为什么是这套

1. 覆盖完整：调度、采集、清洗、质量、分析、服务全链路齐全。  
2. 商业友好：主栈许可证对商用和二开相对友好。  
3. 与现状兼容：可复用现有 `Crawlab + Pinchtab` 基础设施，减少推翻成本。  
4. 可扩展：从单站点到多平台（YouTube/Reddit/X）可平滑扩容。  

---

## 6. 许可证与法务注意点

1. `Valkey` 优先于 Redis 新版本（Redis 许可策略更复杂）。  
2. `SeaweedFS` 作为对象存储默认选型（`MinIO` 主仓库已归档，不作为新建默认选型）。  
3. 若使用 Grafana（AGPLv3），需要结合部署与分发模式做合规评估。  
4. 所有第三方组件引入前，执行一次许可证清单审查（SBOM + License Check）。

---

## 7. 外部数据源约束（必须纳入架构）

1. YouTube Data API：默认每日配额受限，需要预算与配额治理。  
2. Reddit Data API：商业用途需严格遵循条款，必要时签约。  
3. X API：按量计费，必须设置成本阈值和自动降级策略。  
4. 平台数据抓取以官方 API 为主，网页抓取仅作补充与证据链。

---

## 8. 平台落地最小闭环（推荐顺序）

1. 固化统一任务模型：`crawl_job -> fetch_task -> normalize_task -> quality_task -> publish_task`。  
2. 建立 Worker 服务：`HTTP Worker`、`Browser Worker`、`Normalize Worker`、`Quality Worker`，由队列驱动。  
3. 建立三层数据：Raw（对象存储）/ Silver（结构化）/ Gold（服务表）。  
4. 建立幂等机制：`source_id + normalized_url + fetch_window`。  
5. 建立质量门槛：覆盖率、置信度、新鲜度达标才发布。  
6. 输出 API 契约：面向 TanStack Table 的字段、筛选、排序、分页统一版本化。  
7. 脚本边界：`collect.mjs/clean.mjs` 不进入生产调度，仅用于一次性回填/对账。  

脚本退役验收（摘要）：
- 连续 30 天生产环境 0 次脚本直调。  
- 四类 Worker（`fetch/normalize/quality/publish`）覆盖率 100%。  
- 脚本仅保留归档与应急入口，不再挂接生产编排。  

---

## 9. 最终结论

在“人手充足 + 长期运营”前提下，推荐直接采用本方案作为平台版基线：  
`Airflow + Crawlab + Pinchtab + Valkey/Kafka + SeaweedFS + PostgreSQL + ClickHouse + OpenSearch + dbt + GE + OTel/Prometheus`。

这套方案兼顾：
- 开源可商用  
- 长期可扩展  
- 与现有项目兼容  
- 支撑收费产品的工程稳定性
