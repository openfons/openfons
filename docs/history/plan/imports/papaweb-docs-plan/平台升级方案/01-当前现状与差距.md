# 当前现状与差距

> 本文只基于仓库真实代码，不只看文档。

## 1. 前端现状

当前没有真正接入工作区、可直接运行的平台前端。

- `apps/web` 仍是占位目录
- 根工作区没有完整的前端 `dev/build` 脚本
- `services/web-console` 下存在 `upstream-tanstack-table`，但它目前更像上游参考代码，不是已经接入本项目的平台前端

这意味着：

- 目前没有平台级入口页
- 目前没有用户任务下达页
- 目前没有统一的后台菜单与页面骨架

## 2. 当前 Query API 现状

现有 API 主要集中在 `services/query-api/src/index.mjs`，当前只提供：

- `GET /healthz`
- `GET /api/v1/tools`
- `GET /api/v1/tasks`
- `GET /api/v1/metrics`
- `GET /api/v1/source-sites`
- `GET /api/v1/exports/tools.csv`
- `GET /api/v1/exports/tasks.csv`
- `POST /api/v1/pipeline/seed-run`

这说明当前 API 更像“底层查询与触发接口”，而不是“平台后台接口”。

## 3. 当前数据库模型现状

当前 schema 已包含这些关键表：

- `crawl_jobs`
- `crawl_tasks`
- `url_state`
- `seed_state`
- `source_sites`
- `policy_versions`
- `quality_scores`
- `gold_tool_entities`
- `pipeline_events`
- `source_run_state`

这说明已有基础：

- 任务链路基础数据已存在
- 质量策略版本化已有基础
- 来源站点配置已有基础
- 机器执行事件已有基础

但当前仍缺少平台对象：

- 一级 `run`
- 用户
- 角色权限
- 审计日志
- 任务模板
- 计划任务
- 导出任务
- 清洗规则集

## 4. 当前执行链路现状

当前主链路已经能跑通：

`seed/discovery -> fetch -> normalize -> quality -> publish`

对应服务包括：

- `services/worker-seed`
- `services/worker-discovery`
- `services/worker-fetch-http`
- `services/worker-normalize`
- `services/worker-quality`
- `services/publish_worker`

判断：

- 执行面已经存在
- 控制面仍然很薄

## 5. 当前三层策略现状

你们之前说的三层逻辑，在代码里已经存在真实实现。

当前策略定义位于：

- `apps/crawler/src/three-layer-runner.mjs`

当前三层包括：

- `aggressive`
- `conservative`
- `fallback`

当前每层策略已有这些字段：

- `pinchtabMode`
- `requiresPinchtab`
- `maxFailures`
- `maxConsecutiveFailures`
- `blockCountThreshold`
- `blockRateThreshold`
- `headroom`

判断：

- 三层策略不是空想，代码里已经有
- 但它目前还是 CLI 脚本能力
- 还不是前端可配置的平台能力

## 6. 当前清洗逻辑现状

清洗逻辑位于：

- `apps/crawler/src/clean.mjs`

当前清洗逻辑核心特征包括：

- 标题归一化
- URL 去跟踪参数
- 优先选择官方链接
- 按 `official_domain` 优先去重
- 其次按标题和来源去重
- 保留质量更高的一条

判断：

- 清洗能力已经有了
- 但还没有平台化
- 还不能由前端用户通过规则集去配置和预览

## 7. 当前平台的主要缺口

仓库当前明显缺少：

- 登录体系
- 用户体系
- 角色权限体系
- 审计日志体系
- 一级 `run` 对象
- 任务模板
- 计划任务
- 正式的三层策略配置 API
- 正式的清洗规则配置 API
- 导出任务中心
- 原始证据与产物中心

## 8. 最终结论

当前项目不是“什么都没有”，而是：

- 执行链路已经有
- 数据底座已经有
- 最小查询接口已经有
- 平台控制层和前端入口还没有

所以这次升级的本质不是重做采集系统，而是把已有能力收敛成一个正式可操作的平台。
