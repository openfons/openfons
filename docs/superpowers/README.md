# Superpowers Working Docs

`docs/superpowers` 保存的是工作过程中的设计稿、实现计划和执行辅助文档，不等同于当前正式归档目录本身。

统一口径如下：
- 如果历史文档里出现 `docs/plan1/...`、`docs/references/...` 或 `docs/plan/...`，先判断它是在描述“当时的落地位置”还是“迁移动作本身”。
- 当前正式目录以现行结构为准：
  `docs/sot` 放跨阶段 Source of Truth；
  `docs/workbench` 放当前架构、案例、机会与页面规划；
  `docs/history` 统一收口旧规划、阶段验证、上下文和原始输入；
  `docs/superpowers` 继续保留过程设计与执行记录。
- 若历史计划与当前正式目录不一致，以正式目录中的现行文件位置为准，不以旧计划中的历史路径为准。

阅读建议：
- 看“当前正式文档”时，优先进入 `docs/sot`、`docs/workbench`、`docs/history`。
- 看 `docs/superpowers` 时，把它理解为“当时怎么设计、怎么执行、怎么迁移”的过程记录。
