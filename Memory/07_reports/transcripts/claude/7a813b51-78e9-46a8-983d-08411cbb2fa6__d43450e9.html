Here's a technical breakdown of Camoufox based on the README:

## Architecture

Camoufox is a custom Firefox build, not a wrapper. Fingerprint data is "intercepted at the C++ implementation level, making the changes undetectable through JavaScript inspection." It uses Juggler (Firefox's automation protocol, distinct from CDP) rather than CDP, and patches Juggler to give Playwright an isolated "copy" of the page — the real page is unaffected and can't detect reads or listeners.

## Fingerprint Spoofing

All spoofing happens at the browser engine level, so hijacked properties appear native. Covered properties include:

- Navigator (device, OS, hardware, browser, locale)
- Screen size, resolution, viewport, window dimensions
- WebGL parameters, extensions, context attributes, shader precision
- WebRTC IP (at protocol level)
- AudioContext (sample rate, output latency, channel count)
- Geolocation, timezone, locale, Intl
- Battery API, device voices, speech playback rate
- Microphone/webcam/speaker counts
- Network headers (User-Agent, Accept-Language) matched to navigator

Font fingerprinting is countered by bundling Windows/Mac/Linux system fonts and randomly offsetting letter spacing.

Fingerprint distribution uses BrowserForge to mimic real-world market share stats (e.g., Linux spoofed ~5% of the time, matching actual traffic ratios).

## Stealth / Anti-Detection

- `navigator.webdriver` is patched to not expose automation
- Playwright's `window.__playwright__binding__` and init scripts run in an isolated scope, invisible to the page
- Headless mode is patched to appear identical to a normal window
- Pointer type headless detection is fixed
- Fission content isolation re-enabled
- PDF.js re-enabled (its absence is a detection signal)
- Human-like mouse movement algorithm (rewritten in C++ from HumanCursor, with distance-aware trajectories)

## Playwright Integration

Drop-in replacement for existing Playwright code — only browser initialization changes:

```python
from camoufox.sync_api import Camoufox
with Camoufox() as browser:
    page = browser.new_page()
```

Async API also supported via `AsyncCamoufox`.

## Memory & Performance

Described as using ~200MB memory, faster than stock Firefox due to stripping Mozilla services, telemetry, and CSS animations. Optimizations pulled from FastFox, LibreWolf, PeskyFox, and Ghostery patches.

## Packages

- `camoufox` (PyPI) — stable releases
- `cloverlabs-camoufox` — alpha releases with per-context fingerprints and hardware spoofing; updated more frequently

## Limitations

- Does not fully support injecting Chromium fingerprints — some WAFs test for SpiderMonkey engine behavior which can't be spoofed
- Fingerprints must be internally consistent; inconsistencies (e.g., Windows UA + Apple M1 GPU) will be flagged
- Human-like mouse movement "isn't perfect" and may be detected by sophisticated analysis
- As of 2026, there was a maintenance gap causing performance regression due to "newly discovered fingerprint inconsistencies" — active development resumed

## Build System

Linux-only build (WSL not supported). Pipeline: fetch Firefox source → apply patches → build → package for Linux/Windows/macOS. Based on the LibreWolf build system. Docker support is available via a `Dockerfile` in the repo.

## Camoufox Technical Summary

### Architecture

Camoufox is built on Firefox rather than Chromium. The rationale given includes:

- "Chrome is bundled with certain features that Chromium does not have. Anti-bot providers can detect if you are using Chromium rather than Chrome."
- CDP (Chrome DevTools Protocol) is a more common bot-detection target; Juggler, Firefox's automation layer, "operates on a lower level than CDP, making it less prone to JS leaks."
- Firefox has more established fingerprinting-resistance research than Chromium.

The critical architectural distinction is that fingerprint modification happens at the C++ implementation level, not via JavaScript injection. This avoids the detectable DOM tampering that most other stealth tools rely on. "All of Juggler's internal Page Agent Javascript is ran in a sandboxed world," making Playwright's presence extremely difficult to detect.

Source availability: as of v146.0.1-beta.25, all source is public. Earlier builds (v135.0.1-beta.24 and below) include a closed-source Canvas patch, though you can build without it.

---

### Fingerprint Parameters Controlled

The page lists:

- All navigator properties (device, OS, hardware, browser)
- Screen size, resolution, window, and viewport properties
- Geolocation, timezone, locale, and `Intl` object spoofing
- WebRTC IP spoofing at the protocol level (not via JS)
- Speech synthesis voices and playback rate
- WebGL parameters, supported extensions, context attributes, and shader precision formats
- Font spoofing with anti-fingerprinting measures

---

### WebGL / Canvas

- WebGL is addressed at the parameter and extension level, including context attributes and shader precision formats.
- Canvas has a dedicated patch (closed-source in older releases, open-source as of v146.0.1-beta.25). The page notes the "official GitHub releases for v135.0.1-beta.24 and lower do have a closed source Canvas patch."
- AudioContext is not explicitly detailed in the provided content.

---

### Playwright Integration

Camoufox integrates via Playwright through Juggler. Because Juggler runs its agent scripts in a sandboxed context and operates below the level of CDP, it is described as making it "nearly impossible for a page to detect the presence of Playwright." No JS is injected into the main world's DOM.

---

### Statistical Fingerprint Distribution

Rather than using a static fake profile, Camoufox uses BrowserForge to rotate device characteristics (screen, OS, hardware) to match real-world statistical distributions. The explicit design goal is to avoid thousands of requests sharing identical fingerprints, which itself signals bot traffic.

---

### Performance

Described as "debloated & optimized for memory efficiency." No specific benchmarks are provided in the introduction page.

---

### Human-like Behavior

Includes a natural mouse movement algorithm and blocks/circumvents ads. CSS animations are disabled for speed.

---

### Python Package / API

A PyPI package (`camoufox`) handles updates and automatic fingerprint injection. The "Get started" entry point is the `python/` section of the docs.

---

### Headless Mode, Docker, and Detailed API Options

These are not covered in the provided introduction page. You would need to check the Python integration docs or the full API reference sections of the Camoufox site for those details.

---

### Comparison with Other Stealth Tools

The page explicitly critiques the dominant approach: "Many modern 'hardened' browsers and extensions often become counterproductive by making users more distinguishable through detectable JavaScript tampering, blocking certain web APIs, and using static device information." Camoufox's counter-approach is lower-level C++ patching, statistical distribution mimicry, and zero main-world JS injection.

Here is a technical summary of Camoufox usage based on the provided documentation:

---

## Basic Integration

Camoufox is a drop-in Playwright replacement. Only the browser initialization changes:

**Sync:**
```python
from camoufox.sync_api import Camoufox

with Camoufox() as browser:
    page = browser.new_page()
    page.goto("https://example.com")
```

**Async:**
```python
from camoufox.async_api import AsyncCamoufox

async with AsyncCamoufox() as browser:
    page = await browser.new_page()
    await page.goto("https://example.com")
```

All standard Playwright Firefox launch options are also accepted.

---

## Headless Mode

Controlled via the `headless` parameter (`Optional[Union[bool, Literal['virtual']]]`):

- `headless=True` — standard headless mode
- `headless="virtual"` — uses Xvfb virtual display (Linux only)
- Default is `False` (headed)

---

## Persistent Context / Sessions

```python
with Camoufox(
    persistent_context=True,
    user_data_dir='/path/to/profile/dir',
) as context:
    ...
```

Requires `user_data_dir`. This enables browser state persistence across sessions.

---

## Performance & Memory

- **Image blocking:** `block_images=True` blocks all image requests; the docs note this "can help save your proxy usage."
- **Cache:** Disabled by default because "it uses more memory." Disabling cache also prevents `page.go_back()` / `page.go_forward()`. Enable with `enable_cache=True`.

---

| Parameter | Type | Purpose |
|---|---|---|

| `fonts` | `Optional[List[str]]` | Additional system fonts to load |

## Integration Patterns

**Proxy + GeoIP matching** (prevents proxy detection):
```python
with Camoufox(geoip="203.0.113.0", proxy=...) as browser: ...
# or auto-detect:
with Camoufox(geoip=True, proxy=...) as browser: ...
```

**Cloudflare Turnstile (requires `disable_coop`):**
```python
with Camoufox(disable_coop=True, window=(1280, 720)) as browser:
    page = browser.new_page()
    page.goto('https://nopecha.com/demo/cloudflare')
    ...
```

**Window size:** The docs warn that a fixed window size "can lead to fingerprinting" and should only be used for debugging.

**WebGL config** must match the target `os`, or it will cause fingerprint leaks.
