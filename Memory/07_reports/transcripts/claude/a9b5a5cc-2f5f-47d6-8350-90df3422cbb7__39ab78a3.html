8. _config_utils.py lines 115-134 -- save_config does not escape special characters in TOML values

lines.append(f'path = "{project.path}"')

If project.path contains backslashes (common on Windows, e.g. C:\Users\...) or double quotes, the resulting TOML will be malformed. Backslashes in TOML double-quoted strings are escape characters. Windows paths will be corrupted on round-trip (e.g., C:\Users\new becomes C:<TAB>Users<newline>ew when the TOML is re-read). The values should use TOML literal strings (single quotes) or properly escape backslashes.

Note: register_project() stores paths using .as_posix() (forward slashes), which avoids this for projects registered through that function. But if a user manually edits config.toml with Windows paths or if cfg.watch_dirs contains backslash paths, the round-trip through save_config + load_config will corrupt them.
