## SKILL.md Format Overview

Every skill needs a SKILL.md file with two parts: YAML frontmatter (between --- markers) that tells Claude when to use the skill, and markdown content with instructions Claude follows when the skill is invoked.

## Required Structure

The SKILL.md contains the main instructions and is required. This file must start with a YAML frontmatter to hold name and description fields, which are required metadata.

## YAML Frontmatter Fields

The essential frontmatter includes:
- The name field becomes the /slash-command, and the description helps Claude decide when to load it automatically.
- The description is critical for skill selection: Claude uses it to choose the right Skill from potentially 100+ available Skills. Your description must provide enough detail for Claude to know when to select this Skill, while the rest of SKILL.md provides the implementation details.

## Supporting Files

Other files are optional and let you build more powerful skills: templates for Claude to fill in, example outputs showing the expected format, scripts Claude can execute, or detailed reference documentation. Reference these files from your SKILL.md so Claude knows what they contain and when to load them.

skill-name/ ├── SKILL.md (required) │ ├── YAML frontmatter (name, description required) │ └── Markdown instructions └── Bundled Resources (optional) ├── scripts/ - Executable code for deterministic/repetitive tasks ├── references/ - Docs loaded into context as needed └── assets/ - Files used in output (templates, icons, fonts)

For the complete official documentation, visit the [Claude Code Skills documentation](https://code.claude.com/docs/en/skills).

## Skills Directory Structure

Every Skill is defined in a markdown file named SKILL.md (case-insensitive) with optional bundled files that's stored under /scripts, /references, and /assets.

The standard structure for a skill is:

skill-name.zip containing SKILL.md at the root level, along with optional references/, scripts/, assets/, and templates/ directories

Only SKILL.md is required, though the optional directories provide additional capabilities.

## Core Components

**SKILL.md File:**
SKILL.md is a markdown file that follows a two-part structure - frontmatter and content, where the frontmatter configures HOW the skill runs (permissions, model, metadata), while the markdown content tells Claude WHAT to do.

**Supporting Files:**
Optional files let you build more powerful skills: templates for Claude to fill in, example outputs showing the expected format, scripts Claude can execute, or detailed reference documentation.

## Directory Purposes

- **scripts/**: Use scripts/ for complex multi-step operations, data transformations, API interactions, or any task requiring precise logic better expressed in code than natural language.

- **references/**: Put details in references/ etc. for documentation and knowledge bases.

- **assets/**: These bundled files can be Python scripts, shell scripts, font definitions, templates, etc.

- **templates/**: Used for storing templates that Claude can fill in during skill execution.

## Frontmatter Format

Frontmatter includes YAML configuration with fields like name, description, allowed-tools, and version.

## Official SKILL.md Format

Every skill needs a SKILL.md file with two parts: YAML frontmatter (between --- markers) that tells Claude when to use the skill, and markdown content with instructions Claude follows when the skill is invoked.

### Required Frontmatter Fields

The file must start with a YAML frontmatter to hold name and description fields, which are required metadata.

**Example structure:**

```yaml
---
name: skill-name
description: Clear description of what the skill does and when to use it.
---
# Skill Title

Detailed instructions and guidelines...
```

### Field Specifications

- **`name:`** Must use lowercase letters, numbers, and hyphens only.

- **`description:`** Critical—Claude uses this to determine when to invoke your Skill (200 characters maximum).

### Optional Supporting Files

Other files are optional and let you build more powerful skills: templates for Claude to fill in, example outputs showing the expected format, scripts Claude can execute, or detailed reference documentation. Reference these files from your SKILL.md so Claude knows what they contain and when to load them.

The standard skill structure includes optional directories for `scripts/`, `references/`, and `assets/` alongside SKILL.md.

## Directory Location and Structure

Personal skills for Claude Code should be added to `~/.claude/skills/`, while project-specific skills go in `.claude/skills/`.

## Automatic Discovery

Skills defined in `.claude/skills/` within directories are loaded automatically and picked up by live change detection, so you can edit them during a session without restarting. Additionally, nested skills discovery in Claude Code automatically finds skills in subdirectory `.claude/skills` folders when working with files, introduced in v2.1.6.

## Skill Structure

Custom Skills are created as directories with SKILL.md files that Claude discovers and uses automatically. Every skill needs a SKILL.md file with two parts: YAML frontmatter (between --- markers) that tells Claude when to use the skill, and markdown content with instructions Claude follows when the skill is invoked.

## Installation Methods

For Claude Code, skills can be added to `~/.claude/skills/` (personal) or `.claude/skills/` (project) by cloning the GitHub repository and copying the skill folder to your chosen directory.

## Nested Skills Discovery

When working with files in a subdirectory, Claude automatically discovers and loads skills from that subdirectory's `.claude/skills` folder, with skills in nested directories discovered based on your current file context.

## Official Claude Code Skill Format and Structure

Based on official Anthropic documentation and your local skill implementation, here's the complete specification:

### 1. Required Directory Structure

skill-name/
├── SKILL.md (REQUIRED)
├── scripts/ (optional)
├── references/ (optional)
├── assets/ (optional)
└── agents/ (optional)

### 2. Required Files

SKILL.md is the ONLY required file. It contains:
- YAML frontmatter with metadata (at the top, between --- markers)
- Markdown instructions that Claude follows when the skill is invoked

### 3. SKILL.md Format Specification

Structure:

---
name: skill-name-in-lowercase
description: Clear, specific description (200 characters max) of when Claude should use this skill
---

# Skill Title

Detailed markdown instructions for Claude to follow...

Required Frontmatter Fields:
- name: Lowercase letters, numbers, hyphens only. Becomes the /slash-command
- description: Critical metadata. Claude uses this to decide when to automatically load your skill. Must be specific enough for Claude to select the right skill from 100+ possibilities (200 characters max)

Optional Frontmatter Fields:
- Version information
- Allowed tools/permissions
- Other configuration metadata

### 4. Supporting Directory Purposes

scripts/ (optional)
- Executable code for deterministic/repetitive tasks
- Python scripts, shell scripts, or other runnable code
- Claude executes these for complex multi-step operations, data transformations, API interactions
- Referenced from SKILL.md when needed

references/ (optional)
- Documentation and knowledge bases
- Detailed guides, API documentation, configuration specs
- Claude loads these into context as needed when SKILL.md references them
- Keep implementation details here to avoid cluttering the main instructions

assets/ (optional)
- Files used in output: templates, icons, fonts, example outputs
- Subdirectories like templates/ for template files
- Language variants like assets/templates/en/, assets/templates/zh-cn/

agents/ (optional)
- Configuration files for sub-agents (e.g., openai.yaml)
- Used for multi-agent skill orchestration

### 5. Skill Installation and Discovery

Installation Locations:

| Location | Scope | Auto-discovered |
|----------|-------|-----------------|
| ~/.claude/skills/ | Global (all projects) | Yes |
| .claude/skills/ | Project-specific | Yes |
| Nested .claude/skills/ | Subdirectory-scoped | Yes (v2.1.6+) |

Discovery Mechanism:
- Claude automatically discovers skills in .claude/skills/ directories
- Live change detection allows editing skills during sessions without restarting
- Nested skills discovery finds skills in subdirectory .claude/skills/ when working with files in those subdirectories

### 6. Best Practices from Official Guidelines

Description Quality:
- The description is critical—Claude uses it to choose the right skill from potentially 100+ available
- Provide enough detail so Claude knows when to select this skill
- Rest of SKILL.md provides implementation details

File Organization:
- Keep SKILL.md focused on high-level instructions
- Move complex details to references/ files
- Reference bundled resources from SKILL.md so Claude knows what they contain and when to load them

Supporting Resources:
- Only include resources that Claude will actually need
- Use references for documentation, scripts for deterministic operations
- Example: Your memorytree-workflow skill uses 10+ reference files for configuration details

### 8. How Skills Are Used by Claude

1. Claude reads the description field to decide whether to use your skill
2. When invoked (via /slash-command or automatically), Claude loads the full SKILL.md
3. Claude reads referenced references/ files as needed for detailed context
4. Claude executes scripts/ when deterministic operations are required
5. Claude fills in or uses assets/ templates for output
