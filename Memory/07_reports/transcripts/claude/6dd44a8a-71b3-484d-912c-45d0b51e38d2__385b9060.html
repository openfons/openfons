## Round 2：战略 & 投资者视角问题

#### CRITICAL

16. 收费模型与市场定位冲突
- 模型A（开源免费）：GitHub + Markdown 存储，无锁定，用户可能永不付费
- 模型B（SaaS付费）：$9.9/月个人版，$29/月团队版（5人）
- 核心问题：模型A已经满足 80% 用户需求（「我只要记住对话就行」），SaaS 版本的增值点是什么？
- 「云同步」？ → 用户本地存在 Git 仓库，他们已经可以用 iCloud/OneDrive/Dropbox 同步
- 「向量检索」？ → SQLite FTS5 加上简单 UI 就够用
- 「私有化」？ → 开源用户已经可以自部署 Gitea
- 财务硬伤：预计付费转化率会极低（<1%），难以支撑团队
- 对标失败：Obsidian（$10/月，文本编辑生态）能成功，但 OpenMnemo 的增值点更薄弱

- 没有提及用户反馈、用户采访、或任何 PMF 指标

- 如果项目仅有 GitHub Stars 和 NPM 下载，但无实际活跃用户案例，VC 会认为「这是个很好的开源工具，但未必是好的商业模式」

18. 竞争护城河快速崩解的时间表
- 假如 OpenAI/Google/Anthropic 看到 OpenMnemo 的成功，最快多久能复制？
- 答：1-3个月（在主产品里加「导出到 GitHub」按钮）

1. 产品体验 → OpenMnemo CLI + 看板
2. 社区 → 很容易被大厂的品牌压制
3. 数据所有权 → 大厂也能声称支持用户所有权
4. 开源 → 不是防御手段，反而加速被大厂吸收
- 存活概率低：这种模式最终被收购的概率 > 独立上市的概率

- 融资陷阱：开源项目融资常见的路：Seed → 失败或微增长 → 被迫收购 / 闭源
