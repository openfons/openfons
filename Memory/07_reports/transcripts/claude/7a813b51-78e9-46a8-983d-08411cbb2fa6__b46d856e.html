- Header generation matching real browser traffic patterns (0.1-0.2ms per generation)
- Full fingerprint generation: screen properties, navigator data, codecs
- Customizable by browser, OS, device type, locale, HTTP version
- Slim mode for performance-critical evasion
- Direct Playwright injector (async and sync)

Verdict: Essential building block. Both Camoufox and Crawlee use this under the hood. If you're building a custom solution, this is the fingerprint layer you want.

## 4. nodriver (successor to undetected-chromedriver)

GitHub: https://github.com/ultrafunkamsterdam/nodriver

Architecture: Fully async Python library that communicates directly with Chrome DevTools Protocol, bypassing Selenium/WebDriver entirely.

Anti-detection:
- No WebDriver markers (the primary signal WAFs scan for)
- No chromedriver binary needed
- Fresh browser profiles per run, cleaned up afterward
- Headless mode without triggering bot detection
- Built-in tab.cf_verify() for Cloudflare challenges
- Cookie persistence for session reuse
- Expert mode for disabling web security

Verdict: The simplest Python option for quick anti-detection scraping. Less comprehensive than Camoufox (no fingerprint rotation, no deep engine-level spoofing), but much easier to set up. Good for targets with moderate bot protection.

## 5. curl-impersonate -- Network-Level Stealth

GitHub: https://github.com/lwthiker/curl-impersonate

Architecture: Modified curl that replicates real browser TLS and HTTP/2 handshakes at the network level.

Capabilities:
- TLS fingerprint matching for Chrome (99-116), Edge (99-101), Firefox (91-117), Safari (15.3-15.5)
- HTTP/2 settings and pseudo-header ordering matching real browsers
- Compiled with browser-native TLS libraries (NSS for Firefox, BoringSSL for Chrome)

Verdict: Critical for HTTP-level scraping without a browser. Many anti-bot systems fingerprint TLS Client Hello messages. This solves that layer. Combine with BrowserForge headers for a complete HTTP-only stealth stack.

## 6. Scrapy + Anti-Detection Ecosystem (Python)

GitHub: https://github.com/scrapy/scrapy

Scrapy itself is a bare framework, but the ecosystem provides the stealth layers:
- scrapy-playwright (https://github.com/scrapy-plugins/scrapy-playwright) -- Playwright integration for JS rendering, multi-browser support, context isolation
- scrapy-impersonate -- Uses curl-impersonate under the hood for TLS-level browser impersonation
- scrapy-rotating-proxies -- Proxy rotation middleware
- scrapy-zyte-smartproxy -- Commercial proxy integration

Architecture advantage: Scrapy's middleware pipeline makes it easy to compose these layers. You get distributed crawling via Scrapy-Redis or Scrapy-Cluster, plus the mature item pipeline for data processing.

Verdict: Best choice if you need a battle-tested distributed Python pipeline and want to compose your own anti-detection stack from modular middleware. More assembly required than Crawlee, but more flexible.

## 7. MediaCrawler -- Chinese Social Media Specialist

GitHub: https://github.com/NanmiCoder/MediaCrawler

Targets: Xiaohongshu, Douyin, Kuaishou, Bilibili, Weibo, Tieba, Zhihu (7 platforms)

Architecture: Playwright-based browser automation that preserves login sessions and extracts signature parameters via JS expressions, avoiding the need to reverse-engineer each platform's encryption.

Anti-detection:
- Browser context preservation (login state caching)
- IP proxy pool integration
- Rate limiting controls
- User-agent management via Playwright

Data: Keyword search, post retrieval, nested comments, creator profiles. Export to CSV/JSON/JSONL/Excel/SQLite/MySQL.

Limitations: No fingerprint rotation, no TLS-level stealth, no distributed architecture. It's a single-machine tool that relies on looking like a logged-in user in a real browser.

Verdict: The go-to for Chinese social media data collection. But it's not a platform -- it's a collection of per-platform scrapers. For production use, you'd want to wrap it in an orchestration layer and add Camoufox or fingerprint rotation.

## 8. Go-Based Frameworks

Colly (https://github.com/gocolly/colly)
- 1k+ req/sec, clean API, async/parallel scraping, cookie handling, distributed support
- Minimal anti-detection: basic proxy support, no fingerprint management
- Best for: High-throughput HTTP scraping where anti-detection isn't the primary concern

Katana (https://github.com/projectdiscovery/katana) -- by ProjectDiscovery
- Standard + headless crawling modes, JavaScript rendering
- TLS fingerprint randomization (experimental)
- Captcha solving integration (reCAPTCHA, Cloudflare Turnstile, hCaptcha)
- Form auto-filling, technology detection
- Best for: Security recon and crawling JS-heavy sites. The captcha integration is unique in the Go space.

Geziyor (https://github.com/geziyor/geziyor)
- 8.7k req/sec, JS rendering, proxy rotation (HTTP/HTTPS/SOCKS5)
- Multi-level caching (Memory, Disk, LevelDB)
- No anti-detection features
- Best for: Raw speed when you don't need stealth

Verdict: Go frameworks lag behind Python/Node in anti-detection. Katana is the most promising with its TLS randomization and captcha solving, but none match Crawlee or the Python ecosystem for stealth.

## 9. SpiderFlow -- Visual Scraping Platform

Anti-detection: Minimal -- proxy support and cookie management only.

Verdict: Good for non-technical users or rapid prototyping. Not suitable for anti-detection-heavy scenarios.

## 10. Browserless -- Headless Browser as a Service

GitHub: https://github.com/browserless/browserless

Architecture: Dockerized headless browser service (Chrome, Firefox, WebKit) with WebSocket API for Puppeteer/Playwright.

Open-source tier: Session management, concurrency control, debug viewer, crash resilience.
Premium tier: BrowserQL for anti-detection, captcha solving, fingerprint randomization, residential proxy rotation.

Verdict: Great infrastructure layer if you want to separate browser management from scraping logic. The open-source version is a solid foundation; the anti-detection features are paywalled.

## Recommended Architecture for Your Use Case

Given your papaweb project already has a worker-based pipeline with orchestration, the pragmatic approach:

1. Use Crawlee's PlaywrightCrawler with Camoufox as the browser engine for your worker-fetch-http layer -- this gives you the strongest anti-detection stack with minimal custom code.

2. Use BrowserForge + curl-impersonate for lightweight HTTP-only fetches where you don't need a full browser (sitemap parsing, API calls).

3. Borrow MediaCrawler's platform-specific login/session patterns for Chinese social media targets, but run them inside your existing worker orchestration rather than using MediaCrawler as a standalone tool.

4. Katana is worth watching for its Go-native captcha solving if you ever want to move fetch workers to Go for performance.
