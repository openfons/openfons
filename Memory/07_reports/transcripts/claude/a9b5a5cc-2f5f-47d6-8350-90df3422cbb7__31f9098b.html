    26→## What It Does
    27→
    28→MemoryTree gives AI coding assistants (Claude Code, Codex, Gemini CLI) a structured memory layer that survives across sessions:
    29→
    30→| | Feature | Description |
    31→|---|---------|-------------|

    36→| :inbox_tray: | **Transcripts** | Import and archive AI chat history from Codex, Claude Code, and Gemini CLI |

    68→## Architecture
    69→
    70→### Per-Repository Structure
    71→
    72→After initialization, your repo will have:
    73→
    74→```
    75→AGENTS.md                    # AI behavior rules for this repo
    76→Memory/
    77→  01_goals/                  # Versioned project goals
    78→  02_todos/                  # Task lists bound to goals
    79→  03_chat_logs/              # Append-only session logs
    80→  04_knowledge/              # (optional) Durable project notes
    81→  05_archive/                # (optional) Retired goals/todos
    82→  06_transcripts/            # (optional) AI chat history archive
    83→    clean/                   #   Cleaned Markdown indexes
    84→    manifests/               #   Import metadata
    85→    raw/                     #   Raw transcript mirrors
    86→```

   108→## Features
   109→
   110→### Memory Management
   111→
   112→| Memory Type | Behavior |
   113→|-------------|----------|
   114→| **Goals** | Changed only after explicit user confirmation of a scope or requirement change. Versioned. |
   115→| **Todos** | Kept in sync with the active goal. Updated when progress changes or next task becomes clearer. |
   116→| **Chat Logs** | Append-only. Never rewritten or deleted. |
   117→| **Knowledge** | Stores architecture decisions, product ideas, operating constraints. |
   118→| **Archive** | Stale context moved here to keep active files concise. |
   119→
   120→### Multi-Client Transcript Import
   121→
   122→Import AI chat history from three clients:
   123→
   124→| Client | Source |
   125→|--------|--------|
   126→| **Codex** | `~/.codex/sessions/**/rollout-*.jsonl` |
   127→| **Claude Code** | `~/.claude/projects/<project>/*.jsonl` |
   128→| **Gemini CLI** | Saved chats or checkpoints under `~/.gemini/` |
   129→
   130→Key principles:
   131→
   132→- Raw transcripts preserved unchanged as source of truth
   133→- Cleaned Markdown generated by **deterministic code** (not model tokens)
   134→- Current repo only stores transcripts belonging to this project; others go to global archive only
   135→- First import asks once whether raw transcripts may be committed to the repo
   136→
   137→### Cross-Project Search
   138→
   139→When searching all memories or chat history:
   140→
   141→1. Query `search.sqlite` for fast full-text lookup
   142→2. Use `sessions.jsonl` for session-level metadata filtering (client, project, date range)
   143→3. Load matching cleaned Markdown for context; confirm exact wording against raw transcripts
   144→4. Combine with the current repo's `Memory/` files
   145→5. Fall back to scanning `clean/` directory if the index does not exist
   146→
   147→### Git Safety
   148→
   149→MemoryTree follows strict isolation rules to never interfere with your repo's workflow:
   150→
   151→| Rule | Description |
   152→|------|-------------|
   153→| File isolation | Only stages/commits `Memory/**` and `AGENTS.md` — never product code |
   154→| Branch isolation | Uses dedicated branch `memorytree/<scope>-<date>-<slug>` |
   155→| Commit format | `memorytree(<scope>): <subject>` or repo-compatible equivalent |
   156→| PR isolation | Separate PR for MemoryTree-only changes |
   157→| Auto-merge | Only when repo still enforces its review and CI checks |
   158→| Auto-push | On by default; skips if no remote; fast-forward only |
   159→| Protected branches | Never overrides branch protection or CI rules |
   160→
   161→The skill stops and asks before proceeding in 8 scenarios: mixed product code, AGENTS.md conflicts, unclear target branch, repo forbids bot pushes, CI conflicts, unknown raw transcript permission, cross-project transcripts, or protected branch refusal.
   162→
   163→For the full policy see [`references/git-policy.md`](references/git-policy.md).
   164→
   165→### Sensitive Info Scanning
   166→
   167→During transcript cleaning, the heartbeat scans for API keys, passwords, tokens, and other secrets:
   168→
   169→- Matches are logged as warnings and written to `alerts.json`
   170→- **No automatic deletion or redaction** — the user decides how to handle them
   171→
   172→### Locale Support
   173→
   174→| Feature | Description |
   175→|---------|-------------|
   176→| Templates | English (`en`) and Simplified Chinese (`zh-cn`) |
   177→| Auto-detect | Detects language from repo content first, then system locale |
   178→| Reply language | Follows user language > repo locale > system locale |
   179→| Non-destructive | Never rewrites existing files just to translate them |
   180→
   181→### Environment Adaptation
   182→
   183→MemoryTree does not assume the target repo has a Python environment:
   184→
   185→```
   186→Prefer uv run python → fall back to system python → fall back to direct template scaffolding
   187→```
   188→
   189→Never auto-creates `.venv` or `pyproject.toml` inside the project.

   191→## Background Heartbeat
   192→
   193→> **Note**: `heartbeat.py` and `memorytree-daemon` are contract specifications — code is not yet implemented.
   194→
   195→The heartbeat automates transcript discovery, import, cleaning, commit, and push without consuming model tokens or requiring human intervention.
   196→
   197→**Execution flow**:
   198→
   199→```
   200→Acquire lock → Load config.toml → Iterate projects → Discover → Import
   201→→ Clean → Commit → (auto_push) Push → Release lock → Exit
   202→```
   203→
   204→### Install the daemon
   205→
   206→```bash
   207→memorytree-daemon install
   208→```
   209→
   210→### CLI commands
   211→
   212→| Command                     | Description |
   213→|-----------------------------|-------------|
   214→| `memorytree-daemon install`   | Register the heartbeat with the OS scheduler (cron / launchd / Task Scheduler). |
   215→| `memorytree-daemon uninstall` | Remove the scheduled heartbeat task. |
   216→| `memorytree-daemon run-once`  | Run a single heartbeat cycle immediately. |
   217→| `memorytree-daemon watch`     | Continuous loop for development and debugging only. |
   218→| `memorytree-daemon status`    | Show registration state and last execution result. |
   219→
   220→### Key settings (`~/.memorytree/config.toml`)
   221→
   222→| Setting              | Default  | Description |
   223→|----------------------|----------|-------------|
   224→| `auto_push`          | `true`   | Push to remote after each commit. Skips if no remote is configured. |
   225→| `heartbeat_interval` | `"15m"`  | Interval between heartbeat executions. |
   226→| `log_level`          | `"info"` | Log verbosity (`debug`, `info`, `warn`, `error`). |
   227→
   228→### Design constraints
   229→
   230→- Single execution, stateless, idempotent — safe to re-run at any interval
   231→- One computer = one scheduled heartbeat task
   232→- Will not silently register — always confirms before `install`
   233→
   234→For details see [`references/heartbeat-scheduling.md`](references/heartbeat-scheduling.md) and [`references/global-configuration.md`](references/global-configuration.md).
