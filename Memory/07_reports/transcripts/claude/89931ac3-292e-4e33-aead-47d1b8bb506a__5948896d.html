## 问题描述

在 Windows 上执行 `memorytree-daemon install` 后，每次心跳触发（每 5 分钟）都会弹出一个黑色 cmd 窗口，持续约 1-2 秒后消失，严重干扰用户体验。

## 根本原因

`installSchtasks` 将 `/tr` 参数设为：

\`\`\`
node "C:\...\dist\cli.js" daemon run-once
\`\`\`

Windows 任务计划程序默认以交互式会话运行，直接启动 `node` 会创建一个可见的控制台窗口。

## 影响范围

- 平台：Windows only
- 所有通过 `memorytree-daemon install` 注册心跳的用户

## 修复方案

使用 `wscript.exe /B` + VBScript 包装器静默启动 node：

\`\`\`vbs
Set WshShell = CreateObject("WScript.Shell")
WshShell.Run "node ""C:\...\cli.js"" daemon run-once", 0, False
\`\`\`

- `SW_HIDE (0)` 隐藏窗口
- `wscript.exe /B` 抑制 VBScript 错误弹窗
- VBScript 文件写入 UTF-16 LE BOM，确保中文用户名路径正确处理

## Summary

Doubao (豆包, ByteDance's AI assistant) exports conversations as structured TXT files with a well-defined format. This issue tracks adding first-class support for importing these transcripts into MemoryTree.

## Doubao TXT Format

```
Title: 对话标题
URL: https://www.doubao.com/chat/38416801786598914
Platform: 豆包
Created: 2026-03-16 10:22:17
Messages: 28

User: [2026-03-16 10:22:17]
用户消息内容...

AI: [2026-03-16 15:43:17]
AI 回复内容...
```

**Key properties:**
- Header block: `Title`, `URL` (last path segment → `session_id`), `Created` (→ `started_at`)
- Turn markers: `User: [YYYY-MM-DD HH:MM:SS]` or `AI: [YYYY-MM-DD HH:MM:SS]`
- Multi-line message bodies separated by blank lines
- No tool use events (pure chat)
- No `cwd` or `branch` metadata (external platform)

## Proposed Changes

1. **`src/types/transcript.ts`** — Add `'doubao'` to `Client` union type
2. **`src/transcript/parse.ts`** — Add `parseDoubaoTranscript()` function + update `inferClient()` and `parseTranscript()` dispatch
3. **`src/cli/cmd-import.ts`** — Add `forceRepo: boolean` field to `ImportOptions`
4. **`src/cli/main.ts`** — Add `--force-repo` flag for external imports (bypasses `transcriptMatchesRepo()` check since doubao has no `cwd`)

## CLI Usage

```bash
# Import a single doubao TXT file
memorytree import \
  --source doubao_chat_12345.txt \
  --client doubao \
  --root /path/to/repo \
  --force-repo

# Auto-detect client from filename prefix
memorytree import --source doubao_export.txt --root . --force-repo
```

## Why `--force-repo`

External platform transcripts (doubao, browser exports, etc.) have no `cwd` field, so `transcriptMatchesRepo()` returns `false` and the transcript gets written to the global archive instead of the repo's `Memory/` directory. The `--force-repo` flag overrides this check.

- [ ] `--force-repo` flag forces writing to repo `Memory/` directory
