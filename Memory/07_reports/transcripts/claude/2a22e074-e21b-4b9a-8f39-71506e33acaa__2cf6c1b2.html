文件： .github/workflows/release.yml

替换 changesets/action 为自定义步骤：

name: Release

  push:

    permissions:

      - uses: actions/setup-node@53b83947a5a98c8d113130e565377fae1a50d02f  # v6.3.0
        with:
          node-version: 22
          cache: pnpm
          registry-url: https://registry.npmjs.org

      - run: pnpm install --frozen-lockfile

      - name: Check for changesets
        id: changesets
        run: |
          if ls .changeset/*.md 2>/dev/null | grep -qv README.md; then
            echo "has=true" >> "$GITHUB_OUTPUT"
          else
            echo "has=false" >> "$GITHUB_OUTPUT"
          fi

      - name: Version packages
        if: steps.changesets.outputs.has == 'true'
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
          pnpm version-packages
          pnpm install --no-frozen-lockfile
          git add -A
          git commit -m "chore: release"
          git push
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Publish to npm
        if: steps.changesets.outputs.has == 'false'
        run: pnpm release
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

关键变化：
- 不再使用 changesets/action，不需要 pull-requests: write 权限
- fetch-depth: 0 确保 changeset version 能正确读取 git 历史
- 有 changeset 文件时：自动 version → commit → push（触发下一次 CI 走 publish 路径）
- 无 changeset 文件时：直接 publish
- pnpm install --no-frozen-lockfile 在 version 后重新安装（因为 workspace 版本号变了）
- concurrency key 改为 release-${{ github.ref }} 防止并发

program

  })

## 3. 主流工具都这样做吗？

分两派：

### 有内置升级命令的（少数）：
- gh upgrade - GitHub CLI
- rustup update - Rust
- npm install -g npm - npm 自己
- pnpm add -g pnpm - pnpm 自己

### 没有内置升级命令的（多数）：
- firebase-tools → npm install -g firebase-tools
- vercel → npm install -g vercel
- netlify-cli → npm install -g netlify-cli
- typescript → npm install -g typescript
- turbo → npm install -g turbo

大部分 Node.js CLI 都没有内置升级命令，直接让用户重新执行安装命令。

- openmnemo upgrade - 升级项目配置（现有）
- openmnemo update - 升级 CLI 工具（新增）
- 或者 openmnemo self-update
