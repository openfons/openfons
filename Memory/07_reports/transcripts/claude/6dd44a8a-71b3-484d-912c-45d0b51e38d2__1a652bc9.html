   250→## 七、跨仓库搜索架构
   251→
   252→### 问题背景
   253→
   254→Gitea/GitHub 的原生搜索是**仓库级别**的——用户必须进入某个仓库才能搜索其中内容，无法跨仓库全文检索。
   255→
   256→### OpenMnemo 的解法：统一本地 SQLite 索引
   257→
   258→```
   259→心跳触发（每 5 分钟）
   260→        ↓
   261→扫描所有仓库的 manifests/ 目录
   262→        ↓
   263→新 session → 解析 clean Markdown → 写入本地 SQLite FTS5 索引
   264→        ↓
   265→openmnemo search "认证方案"
   266→        ↓
   267→SQLite FTS5 全文检索 → 跨仓库、跨客户端统一结果
   268→        ↓（未来）
   269→向量嵌入索引 → 语义搜索（"找我讨论过的状态管理方案"）
   270→```
   271→
   272→**SQLite 索引 schema（核心字段）：**
   273→
   274→```sql
   275→CREATE VIRTUAL TABLE sessions_fts USING fts5(
   276→  session_id,
   277→  repo,          -- 所属仓库（@alice/work、@alice-family/household）
   278→  client,        -- codex | claude | gemini
   279→  title,
   280→  content,       -- clean markdown 全文
   281→  started_at,
   282→  tags
   283→);
   284→```
   285→
   286→**关键优势：**
   287→- 完全本地，零网络延迟
   288→- 跨仓库（个人仓库 + 组织仓库全部索引）
   289→- 跨客户端（Claude / Codex / Gemini 统一检索）
   290→- 未来可加 sqlite-vec 实现向量语义搜索

   294→## 八、外部对话导入架构
   295→
   296→### 背景
   297→
   298→OpenMnemo 当前只能捕获 CLI 工具（Claude Code / Codex / Gemini CLI）的本地 JSONL 文件。用户在豆包、ChatGPT、Kimi 等平台积累的**历史对话存量**，以及没有 CLI 工具的网页端对话，需要一套独立的导入通道。
   299→
   300→### 三层导入策略
   301→
   302→```
   303→Layer 1：结构化导出（现在就能做）
   304→  ├── 豆包 TXT       → parseDoubao()
   305→  ├── ChatGPT JSON   → parseChatGPT()
   306→  ├── Kimi / 文心等  → parseKimi() / parseErnie()
   307→  └── openmnemo import --client doubao ./dir/
   308→
   309→Layer 2：AI 辅助解析（通用兜底）
   310→  ├── 任意复制粘贴的对话文本
   311→  ├── 无结构的 HTML 导出
   312→  └── openmnemo import --client generic ./file.txt
   313→      → 送入 AI，自动识别 User/AI 角色边界
   314→
   315→Layer 3：浏览器插件实时捕获（Phase 2）
   316→  ├── 无需导出，实时拦截 DOM
   317→  └── 覆盖 Claude.ai / ChatGPT / Gemini Web 等无导出功能的平台
   318→```
   319→
   320→### 豆包 TXT 格式（已确认可解析）
   321→
   322→豆包导出的 TXT 格式已经高度结构化，约 80-100 行 TypeScript 即可完整解析：
   323→
   324→```
   325→Title: 对话标题
   326→URL: https://www.doubao.com/chat/xxxxx
   327→Platform: 豆包
   328→Created: 2026-03-16 10:22:17
   329→Messages: 28
   330→
   331→User: [2026-03-16 10:22:17]
   332→用户消息内容...
   333→
   334→AI: [2026-03-16 15:43:17]
   335→AI 回复内容...
   336→```
   337→
   338→### `openmnemo import` 命令设计
   339→
   340→```bash
   341→# 单文件导入
   342→openmnemo import --client doubao ./conversation.txt
   343→
   344→# 批量导入整个目录
   345→openmnemo import --client doubao ./chat-memo_61_20260316154801/
   346→
   347→# 通用格式（AI 辅助识别结构）
   348→openmnemo import --client generic ./any-conversation.txt
   349→
   350→# 指定目标仓库和隐私级别
   351→openmnemo import --client chatgpt ./conversations.json \
   352→  --repo work \
   353→  --visibility private
   354→```
   355→
   356→执行后自动走完整归档流程：
   357→```
   358→解析原文 → clean Markdown + manifest JSON
   359→       ↓
   360→SHA256 去重（已导入的跳过）
   361→       ↓
   362→AI 提取标签 + 摘要（可选，--no-ai 跳过）
   363→       ↓
   364→git commit + push
   365→       ↓
   366→SQLite FTS5 索引更新
   367→```

   369→### 各平台导出方式

   371→| 平台 | 导出入口 | 格式 | Parser 难度 |
   372→|------|---------|------|------------|
   373→| **豆包** | 对话页 → 导出 | TXT（结构化） | ⭐ 低 |
