# 项目目标 v008

## 元信息
- 版本: v008
- 创建日期: 2026-04-10
- 上一版本: goal_v007

## 北极星目标
基于 `v007` 的真实环境 smoke 结果，补齐 `crawler execution` 的 operator tooling、runtime diagnostics 与 secret bootstrap 收口层，使团队在真正执行 `youtube / tiktok` smoke 之前，就能对宿主机依赖、`.env_uv`、`OPENFONS_SECRET_ROOT`、route 绑定 secret 和占位/失效材料进行 route-aware 预检，并拿到可执行的修复建议，而不是继续停留在 runbook + 异常字符串排障。

## 范围
- 产出一条面向 `youtube / tiktok` 的 route-aware preflight / diagnostics 主线，覆盖：
  - `yt-dlp / uv / .env_uv / TikTok bridge` 等宿主机依赖探测
  - `pinchtab-token / tiktok-account-main / tiktok-cookie-main / global-proxy-pool` 等 secret 材料探测
  - 缺失、占位、格式无效、候选路径、修复建议等结构化结果
- 将 diagnostics 能力同时收口到：
  - 可复用的后端 core
  - `control-api` 只读诊断接口
  - 本机 operator CLI / bootstrap 入口
- 允许安全 bootstrap：
  - 创建缺失目录
  - 创建受控占位 secret 文件
  - 初始化 `.env_uv`
  - 绝不自动写入真实凭据
- 在 runbook 与真实 smoke 流程中复用 diagnostics 结果，推动 `youtube / tiktok` 再次验证。

## 非目标
- 本轮不扩 `twscrape / PRAW / MediaCrawler` 等新 adapter。
- 本轮不新增配置中心 UI 或写接口。
- 本轮不自动下载或静默安装 `yt-dlp`。
- 本轮不自动获取真实 cookie、账号、代理或 token。
- 本轮不解决平台风控、验证码或长期会话维护。

## 验收信号
- 仓库内存在新的 runtime diagnostics 设计与实现计划文档。
- 存在 route-aware diagnostics 输出，能在不执行真实 crawler 的前提下指出每条 route 的 blocker。
- operator CLI 能输出结构化 JSON，并在需要时安全创建缺失的占位 secret / `.env_uv`。
- `control-api` 能返回对应的只读 diagnostics 结果，供后续平台侧复用。
- `youtube / tiktok` 的下一轮 smoke 失败如果仍存在，也应表现为更窄、更可执行的 blocker，而不是笼统的异常链。
