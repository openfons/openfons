# 项目目标 v011

## 元信息
- 版本: v011
- 创建日期: 2026-04-11
- 上一版本: goal_v010

## 北极星目标
在 `v010` 已完成平台级配置中心读路径、管理接口与 runtime 接入的基础上，收口 `platform plugin config center` 的写路径：为 `PluginInstance` 与 `ProjectBinding` 建立受控、可回滚、带 revision 的 repo-backed mutation 闭环，让配置中心从“只能读和校验”升级为“能安全预览、写入、恢复”的平台能力。

## 范围
- 对齐已完成的写路径实现计划：
  - `docs/superpowers/plans/2026-04-11-config-center-write-path-closure.md`
- 当前主线按四个任务推进：
  - Task 1: `write-path contracts + record-aware loaders`
  - Task 2: `atomic persistence / backups / validation-before-write`
  - Task 3: `control-api` 配置中心写接口
  - Task 4: operator runbook
- 本轮关键边界：
  - 只允许写入 repo-visible config：`config/plugins/**` 与 `config/projects/**`
  - secret value 仍保留在 `~/.openfons/secrets/**`，不进入 repo-visible config
  - 写入必须具备 `dryRun`、revision 检查、锁、原子写、backup 与结构化错误返回
  - `control-api` 仍然是唯一 HTTP 写入口；底层写逻辑收敛在 `packages/config-center`

## 非目标
- 本轮不回退到 `v008` 的真实 `youtube / tiktok` smoke 收尾。
- 本轮不扩到配置中心 UI。
- 本轮不增加 secret value 写接口或外部 vault 集成。
- 本轮不增加 delete endpoint 或通用“任意写文件”接口。
- 本轮不扩到 scheduler、notification、artifact delivery 等无关主线。

## 验收信号
- `PluginInstance` 与 `ProjectBinding` 拥有共享的 write contracts 与 record-aware loader。
- 写路径具备 deterministic revision、optimistic concurrency、repo-local lock、atomic write 与 backup。
- `control-api` 能对配置写入提供 `dryRun` 与 `apply` 两种路径，并返回结构化冲突/无效错误。
- 写前校验不依赖开发机本地环境，测试夹具可显式注入 `secretRoot`。
- operator runbook 能描述预览、应用、回滚三类操作，而不泄露真实 secret value。
