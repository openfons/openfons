# 项目目标 v004

## 元信息
- 版本: v004
- 创建日期: 2026-04-07
- 上一版本: goal_v003

## 北极星目标
将 OpenFons 第一阶段从“采集能力验证”推进到“采集内容分析后生成公开 SEO 决策 HTML 页面”的最小闭环，首发主线锁定 `AI procurement / direct API vs OpenRouter`，产物是面向 Google SEO 和小型 AI coding 团队采购决策的 `direct-api-vs-openrouter.html`。

## 范围
- 复用 `docs/workbench` 中已确认的利基选题、页面优先级和 AI procurement 案例判断。
- 基于当前 AI procurement 采集/结构化证据生成 `ReportView`，再导出 `docs/workbench/generated/direct-api-vs-openrouter.html`。
- 页面必须回答“什么时候直连官方 API，什么时候使用 OpenRouter”，而不是展示采集过程。
- 保留官方来源、社区佐证、费用/地区/路由风险和证据边界，避免把 OpenRouter 写成必然更便宜。
- 保持已有 deck 导出和 OpenClaw 批次报告链路可用，但 OpenClaw 不再是当前公开 SEO 首发主线。

## 非目标
- 本轮不直接把页面升级为完整的 Next.js SEO 站点。
- 本轮不重新发起新一轮线上采集。
- 本轮不一次性抽象成适配所有主题的通用发布系统。
- 本轮不发布没有官方价格/可用性依据的“最低价”或“最优推荐”断言。

## 验收信号
- 仓库内存在重新生成的 `docs/workbench/generated/direct-api-vs-openrouter.html`。
- 页面包含 `Quick Answer`、`Executive Summary`、`Where Direct API Wins`、`Where OpenRouter Wins`、`Hidden Costs and Risks`、`Decision Tree by Team Type`。
- 页面内容是用户可读的采购/路由决策分析，不把采集过程日志作为主体内容。
- 页面保留官方来源、社区佐证、证据边界和“OpenRouter is not automatically cheaper”的风险提示。
- 导出脚本、测试和 typecheck 可通过。
